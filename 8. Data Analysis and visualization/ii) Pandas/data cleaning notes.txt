Raw data problems:
- missing values
- duplicate values
- spelling error
- outliers (extreme values--- other than range)
- incorrect datatypes
- inconsistent formatting (eg. date formats)


Data cleaning: 
   Process of detecting, correcting and handling errors in raw data to make it suitable for analysis or modeling.

1- Data collection and inspection
  * type of data in each column (numerical, categorical, datetime, text)
  * missing/duplicate values
  * datatypes
  * inconsistencies

  Techniques:
  -> summary statistics
  -> Distribution plot
  -> info (datatypes/ non-null count)
  -> frequency (categorical data)
  -> visual inspect (manually analyse)

2- Handling missing values
   * Delete rows/column -> more than 50% of data is missing
   * Imputation -> filling missing values
       - Mean/Median/Mode -> applicable for numerical columns
       - Mode/Unknown -> for categorical columns
       - Forward fill/ Backward fill -> Date time columns, Numerical columns
       - Predictive imputation using ML model


3- Fixing inconsistent formatting
   * Male, m, M, f, F, female -> 2 categories -> 6 categories
      -> Replace, lowercase...
   * Extra space -> _AM, AM -> 2 categories logically should be single category
   * date/currency formatting

4- Handling duplicates
   * Remove duplicate records/rows
   * check for primary key
   * Remove near duplicate records

5- Correcting datatypes
   * Numeric -> int/float
   * Categorical -> object
   * datetime -> datetime

6- Handling outliers
   * Genuine outliers -> eg house price ranges
   * errors
    -> Remove outliers, using IQR, z-score
    -> Domain knowledge
    -> Transforming data

Ordinal Encoding converts categorical data with a natural order (e.g., Low, Medium, High) into ordered numbers (\(0, 1, 2\)), preserving ranking. Label Encoding assigns arbitrary integers (often alphabetically) to categories without inherent order (e.g., Red=0, Blue=1), which can mistakenly imply a ranking.
